# GX教育项目交付管理系统 - 开发环境说明

**版本**：V1.0
**日期**：2026年7月


## 一、环境要求

| 组件 | 版本要求 | 说明 |
|------|----------|------|
| Node.js | 18.x LTS | 前端运行环境（必须使用18.x，不支持20+） |
| Python | 3.11+ | 后端运行环境 |
| MySQL | 8.0 | 数据库（建议使用Docker） |
| 包管理器 | npm 9.x / pnpm 8.x | 前端依赖管理 |


## 二、快速开始

### 2.1 克隆代码

```bash
git clone <仓库地址>
cd gx-project
```

### 2.2 配置环境变量

**后端**：复制 `backend/.env.example` 为 `backend/.env`，修改配置

```bash
cp backend/.env.example backend/.env
```

**前端**：复制 `frontend/.env.development` 为 `frontend/.env.local`

```bash
cp frontend/.env.development frontend/.env.local
```

### 2.3 启动数据库（Docker方式）

```bash
docker run -d \
  --name gx_mysql_dev \
  -p 3306:3306 \
  -e MYSQL_ROOT_PASSWORD=GX2026\!root \
  -e MYSQL_DATABASE=gx_project_dev \
  -e MYSQL_USER=gx_user \
  -e MYSQL_PASSWORD=GX2026\!pass \
  -v ~/gx-mysql-data:/var/lib/mysql \
  mysql:8.0 \
  --character-set-server=utf8mb4 \
  --collation-server=utf8mb4_unicode_ci
```

### 2.4 初始化数据库

```bash
docker exec -i gx_mysql_dev mysql -uroot -pGX2026\!root gx_project_dev < ddl.sql
```

### 2.5 启动后端

```bash
cd backend
python -m venv venv
source venv/bin/activate  # Windows: .\venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --host 127.0.0.1 --port 8000 --reload
```

### 2.6 启动前端

```bash
cd frontend
npm install --legacy-peer-deps
npm start
```


## 三、常见环境问题

| 问题                   | 解决方案                                                     |
| ---------------------- | ------------------------------------------------------------ |
| Node.js 版本过高       | 使用 nvm 切换到 18.x：`nvm use 18`                           |
| `react-scripts` 找不到 | 删除 node_modules 重新安装：`rm -rf node_modules && npm install` |
| 端口被占用             | 修改启动端口：`PORT=3001 npm start` 或 `uvicorn ... --port 8001` |
| Python 包安装失败      | 使用国内镜像：`pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple` |
