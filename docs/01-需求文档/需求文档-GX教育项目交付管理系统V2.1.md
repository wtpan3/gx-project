# 需求文档：高新区"AI+教育"项目交付管理系统 V2.1

**版本**：V2.1
**日期**：2026年7月
**状态**：定稿


## 一、项目概述

### 1.1 项目背景

基于Excel表格管理的教育信息化项目，需要将其迁移至线上系统进行规范化管理。项目涵盖从设备采购、交付实施到验收结算的全生命周期管理。

### 1.2 现状分析

| 现状     | 说明                             |
| -------- | -------------------------------- |
| 管理方式 | Excel表格分散管理                |
| 数据更新 | 手动维护，版本混乱               |
| 协作方式 | 文件传递，无法实时同步           |
| 进度跟踪 | 依赖人工汇总汇报                 |
| 资料归档 | 各阶段文档分散存储，难以统一管理 |

### 1.3 项目目标

- 将Excel内容线上化，实现数据集中管理
- 支持多用户协作，实时更新项目状态
- 项目计划（WBS）驱动任务执行，任务完成时上传佐证材料
- 提供标准模板下载，规范各阶段交付物格式
- 实现文件资料的统一管理、归档与批量下载
- 支持项目周报与校级周报的自动生成与导出
- 通过学校看板实现"一校一档"全景洞察
- 支持多年度项目建设，按年份区分设备与任务
- 提供项目健康度综合评价与全局筛选能力
- 支持项目复盘与经验沉淀


## 二、角色与权限总览

### 2.1 角色定义

| 角色         | 核心职责                     | 权限范围                                                |
| ------------ | ---------------------------- | ------------------------------------------------------- |
| **管理员**   | 系统全面管控                 | 全部模块读写 + 操作日志                                 |
| **项目经理** | 项目全局推进与协调           | 全部模块读写（含系统管理核心功能）                      |
| **校园经理** | 指定学校的项目交付执行与管理 | 全部业务模块读写 + 部分系统管理功能，**项目周报仅只读** |

### 2.2 角色说明

**校园经理**：
- 校园经理是项目团队内部角色，并非校方人员
- 每个校园经理负责一所或多所学校的项目交付工作
- 可查看项目全局数据以了解整体进度、对齐工作节奏
- 可参与学校、设备字典、模板等基础数据的维护
- 项目周报仅只读（由项目经理统一编制）

### 4.3 设备系统字典

> **权限**：管理员、项目经理、校园经理（读写）

#### 4.3.1 功能说明

将所有设备的规格信息作为模板在此统一维护。设备录入时只需选择设备，品牌/型号/参数等自动带入。同时维护该设备的计划总数量和建设年份，作为规划基线。

#### 4.3.2 数据字段

| 字段名   | 类型   | 说明                             | 必填 |
| -------- | ------ | -------------------------------- | ---- |
| 项目名称 | 下拉框 | 从数据字典中维护的项目列表选择   | 是   |
| 建设年份 | 下拉框 | 2026 / 2027 / 2028 ...，默认2026 | 是   |
| 系统名称 | 文本   | 如"AI英语听说系统"               | 是   |
| 设备名称 | 文本   | 如"听说考试专用耳机"             | 是   |
| 品牌     | 文本   | 设备品牌                         | 是   |
| 型号     | 文本   | 设备型号规格                     | 是   |
| 参数     | 文本   | 技术参数说明                     | 否   |
| 设备类型 | 下拉框 | 硬件 / 软件 / 服务               | 是   |
| 单位     | 文本   | 台 / 套 / 个                     | 是   |
| 计划数量 | 数字   | 该设备规格的计划/预估总数量      | 是   |

#### 4.3.3 业务规则
- **项目名称+系统名称+设备名称+建设年份** 四者组合唯一
- 建设年份默认2026，支持多年度扩展
- 建设年份为2027年及以后的设备，**仅在设备系统字典中录入**，设备信息模块不可录入
- 被设备信息引用的字典项不可删除（软删除）
- 设备名称下拉支持模糊搜索
- **计划数量与实际录入数量的关系**：设备信息中录入的该设备各批次数量之和超过计划数量时，系统给出橙色警告提示，但不禁止保存，由项目经理自行判断

#### 4.3.4 功能清单
- 字典列表查看（按项目、系统、建设年份筛选）
- 新增 / 编辑 / 删除字典项（软删除）
- 批量导入（Excel）

### 4.7 项目信息管理

> **权限**：管理员、项目经理

#### 4.7.1 功能说明

管理项目整体的基本信息与状态。项目状态影响部分模块的行为。

#### 4.7.2 数据字段

| 字段名       | 类型         | 说明                                       | 必填 |
| ------------ | ------------ | ------------------------------------------ | ---- |
| 项目名称     | 文本         | 项目全称                                   | 是   |
| 项目编码     | 文本（唯一） | 项目唯一标识                               | 是   |
| 项目开始日期 | 日期         | 项目启动日期                               | 是   |
| 项目结束日期 | 日期         | 计划结束日期                               | 是   |
| 项目整体状态 | 下拉框       | 未启动 / 进行中 / 试运行 / 已验收 / 已结项 | 是   |

#### 4.7.3 业务规则
- 项目状态变更为"已验收"后，自动开启项目复盘入口
- 项目状态变更为"已验收"或"已结项"后，不允许修改交付数据（设备信息、项目计划、培训管理等）
- **项目状态回退**：如果项目状态从"已验收"回退到"进行中"，复盘入口关闭，已保存的复盘数据保留但不可编辑，项目再次验收后可选择"重新填写"或"基于之前内容继续完善"

#### 4.7.4 功能清单
- 项目信息查看与编辑
- 项目整体状态变更
- WBS任务的计划结束时间不得超过项目结束日期（保存时校验）


### 4.8 操作日志

> **权限**：管理员

记录所有增、删、改、导入操作。字段：操作人、模块、操作类型、目标ID、变更前后数据（JSON）、IP地址、操作时间。

**批量操作日志规则**：记录操作人、操作类型（批量导入）、导入文件名称、导入记录数量（成功/失败），不记录每条记录的详细变更JSON。

**功能清单**：日志列表查看（按模块、操作人、时间筛选）、日志导出（Excel）、永久保留。


## 五、全局交互规范

### 5.1 全局筛选器

顶部导航增加**全局筛选栏**，所有业务模块统一响应：

| 筛选维度 | 类型   | 选项                                                         | 默认值 |
| -------- | ------ | ------------------------------------------------------------ | ------ |
| 建设年份 | 下拉框 | 2026 / 2027 / 2028 ...                                       | 全部   |
| 学校     | 下拉框 | 所有学校列表                                                 | 全部   |
| 阶段     | 下拉框 | **统一使用数据字典"关联阶段"作为数据源**：到货验收 / 加电测试 / 校级验收 / 培训 | 全部   |

**筛选逻辑**：全局筛选器与各模块独立筛选为**叠加关系**（AND逻辑），模块独立筛选进一步缩小范围。顶部显示当前生效的筛选条件，用户可快速清除。

### 5.2 列表页搜索规范

所有列表页统一搜索交互：
- **关键词搜索框**：模糊匹配当前列表关键字段
- **高级筛选**：点击展开更多筛选条件
- 搜索框位于列表上方左侧，筛选按钮位于右侧

### 5.3 删除操作规范

所有删除操作增加二次确认弹窗：
- 提示："确定要删除【记录名称】吗？此操作不可恢复"
- 如果该记录被其他数据引用，提示："该记录已被引用，无法删除"

### 5.4 批量操作规范

所有列表页（项目计划、设备信息、培训管理、风险管理）支持批量操作：

- 列表左侧增加复选框，支持全选/取消全选
- 选中记录后，列表顶部出现批量操作栏
- 批量操作选项：
  - **批量更新状态**：选中多条记录，统一变更状态
  - **批量分配责任人**：选中多条记录，统一分配责任人
  - **批量删除**：选中多条记录，统一删除（二次确认）
- 批量操作执行前二次确认
- 批量操作结果反馈（成功/失败条数）

### 5.5 数据下钻规范

所有展示统计数据、列表数据、汇总数据的页面，均需支持下钻能力。

#### 5.5.1 定义

**下钻**：用户点击某个数据元素（数字卡片、列表行、图表节点）后，系统跳转或弹出该数据对应的明细页面，查看更详细的数据。

#### 5.5.2 两种下钻模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **页面跳转** | 点击后从当前页面跳转到新页面 | 仪表板卡片点击 → 跳转到列表页 |
| **弹窗/抽屉** | 点击后在当前页面弹出浮层 | 列表页某一行 → 弹出该记录的详情浮层 |

#### 5.5.3 下钻触发方式

| 元素类型 | 触发方式 | 交互反馈 |
|----------|----------|----------|
| 数字卡片（仪表板） | 点击卡片任意区域 | 鼠标悬停变为手型，点击后跳转 |
| 列表行 | 点击行任意区域 | 鼠标悬停变为手型，点击后弹出详情 |
| 表格中的关联字段 | 点击字段值 | 字段变为蓝色可点击样式，点击后弹出详情 |
| 图表节点 | 点击柱子/折线点/扇形区 | 鼠标悬停变为手型，点击后跳转 |
| 带筛选条件的跳转 | 点击后带参数 | URL 携带筛选参数，列表自动过滤 |

#### 5.5.4 下钻模式选择

| 场景 | 下钻模式 | 说明 |
|------|----------|------|
| 仪表板数字卡片 → 列表页 | **页面跳转** | 需要展示完整列表，支持进一步筛选 |
| 图表点击 → 筛选列表 | **页面跳转** | 带上筛选参数跳转 |
| 列表页行 → 详情页 | **弹窗（Drawer）** | 查看单个记录的完整信息，不离开当前页面 |
| 关联字段点击 → 关联对象详情 | **弹窗（Drawer）** | 如点击学校名，弹出该校详情 |

#### 5.5.5 弹窗/抽屉详情页标准

- **类型**：优先使用 Drawer（抽屉），从右侧滑出
- **宽度**：600px
- **标题**：显示记录名称 + "详情"
- **内容**：分组展示所有字段
- **操作**：底部放置"编辑"、"关闭"等按钮
- **关闭方式**：点击"×"、点击遮罩层、按 ESC 键，均可关闭

#### 5.5.6 下钻跳转目标

| 来源 | 点击元素 | 跳转目标 |
|------|----------|----------|
| 仪表板 - 学校总数 | 数字卡片 | `/schools`（学校列表） |
| 仪表板 - 设备系统数 | 数字卡片 | `/device-systems`（设备系统列表） |
| 仪表板 - 进行中任务 | 数字卡片 | `/project-plan?status=进行中` |
| 仪表板 - 高风险项 | 数字卡片 | `/risk-management?level=高` |

| 仪表板 - 交付进度图表 | 学校进度条 | `/delivery-progress?school=学校ID` |
| 列表页 - 数据行 | 行任意位置 | 该记录的详情抽屉 |
| 列表页 - 关联字段 | 关联字段值 | 关联对象的详情抽屉 |

#### 5.5.7 各模块下钻场景汇总

| 模块 | 下钻来源 | 跳转目标 |
|------|----------|----------|
| 首页仪表板 | 数字卡片、进度条 | 对应模块的列表页（带筛选） |
| 项目计划（WBS） | WBS 树节点 | 任务详情抽屉 |
| 项目计划（WBS） | L1/L2 阶段汇总数 | 该阶段下的任务列表（页面跳转） |

#### 5.5.8 下钻层级限制

| 层级 | 说明 |
|------|------|
| 第一层 | 仪表板 → 列表页 |
| 第二层 | 列表页 → 详情抽屉 |

#### 5.5.9 下钻交互反馈

| 场景 | 交互细节 |
|------|----------|
| 鼠标悬停 | 下钻元素显示下划线或背景色变化，鼠标变为手型 |
| 点击后 | 如果数据量小，瞬间响应；数据量大时，显示加载状态 |

| 空状态 | 如果下钻后无数据，显示"暂无数据"占位，并提供"返回"按钮 |

> **后续开发约定**：各模块开发时，默认支持下钻能力。仪表板卡片 → 页面跳转，列表行 → 抽屉详情。

## 六、业务模块

### 6.1 首页（项目概览）

#### 6.1.1 功能定位

首页是**项目整体情况的总览页面**，让项目经理在打开系统后 30 秒内完成"判断项目状态 → 定位关键问题 → 确认个人待办"的决策闭环。首页只承载"判断依据"，不承载全部细节，明细一律下钻到对应模块查看。

页面自上而下分为五个区域：**顶部卡片区 → 进度区 → 里程碑区 → 项目风险跟踪区 → 待办区**。

#### 6.1.2 顶部卡片区（项目规模总览）

展示 7 个关键规模指标卡片，均支持下钻：

| 卡片     | 含义                     | 数据来源                       |
| -------- | ------------------------ | ------------------------------ |
| 覆盖学校 | 项目总学校数             | 学校表                         |
| 重点学校 | 标记为重点的学校数       | 学校表.is_key                  |
| 系统总数 | 项目系统总数             | 系统表                         |
| 设备类型 | 硬件设备种类数           | 设备字典表（按设备名称去重）   |
| 产线类型 | 生产线种类数             | 产线类型字典表                 |
| 外采设备 | 需外部采购的设备种类数   | 设备字典表（来源=外采）        |
| 硬件总数 | 硬件设备总台数           | 设备信息表                     |

#### 6.1.3 进度区（三栏+整体进度）

**整体进度展示规则**（置顶）：
- 整体进度条展示在进度区顶部，三栏之前
- 默认显示整体进度百分比（如 68%）
- 加权计算公式默认隐藏，鼠标悬停时显示：`= 学校 30%×65% + 硬件 40%×72% + 软件 30%×53%（权重可配置）`

**三栏进度明细**：

| 栏目             | 展示方式   | 说明                                                         |
| ---------------- | ---------- | ------------------------------------------------------------ |
| 学校交付进度     | 环形图     | 6 种状态分布（待启动/装修中/安装中/调试中/培训中/已完成），标注重点学校。环形图尺寸自适应容器 |
| 硬件交付进度     | 环形图     | 5 种状态分布（待发货/已到货/已安装/已调试/运行中）。环形图尺寸自适应容器 |
| 软件平台交付进度 | 进度条列表 | 4 个模块（数智工作台/教育数据指挥中心/智能应用创编平台/应用能力服务）。**进度条颜色与状态对应**：已上线=绿色(#52c41a)、测试中=橙色(#fa8c16)、开发中=蓝色(#1677ff) |

各环形图/进度条支持点击下钻到对应模块的筛选列表。

#### 6.1.4 整体进度计算规则

整体进度由学校、硬件、软件三个维度加权计算：

```
整体进度 = (学校维度权重 × 学校完成率) + (硬件维度权重 × 硬件完成率) + (软件维度权重 × 软件完成率)

权重分配（可配置）：
- 学校维度：30%
- 硬件维度：40%
- 软件维度：30%

学校完成率 = 已完成验收学校数 / 总学校数
硬件完成率 = 已完成验收设备数 / 总设备数
软件完成率 = 已上线功能模块数 / 总功能模块数
```

#### 6.1.5 里程碑区

关键里程碑达成情况，以甘特条形式展示（仅展示影响交付的关键节点，如硬件到货验收、加电测试完成、教师培训完成、校级验收）。

**表结构**：`阶段 | 任务 | 时间轴(甘特) | 开始日期 | 结束日期 | 完成状态`

**列宽规范（固定宽度，不允许内容撑开）**：
- 阶段列：120px
- 任务列：180px
- 甘特图列：占据剩余宽度，最小 400px
- 计划开始时间列：120px（完整显示 yyyy-mm-dd 格式）
- 计划完成时间列：140px（完整显示 yyyy-mm-dd 格式）
- 状态列：100px（完整显示"已完成/进行中/未开始"）

**表头与内容对齐**：全部列（表头 + 内容）均**居中显示**。

**通用表格规则**（适用于首页所有表格：里程碑、项目风险跟踪等）：
- 每一列设置固定宽度，采用 `table-layout: fixed` 强制固定布局，不允许内容自动撑开列宽
- 超出部分用省略号（…）截断，鼠标悬停显示完整内容（title 属性或 Tooltip 组件）
- 表格总宽度超出容器时，给表格容器增加 `overflow-x: auto`，允许横向滚动查看完整内容，而非压缩列宽或强制换行

**时间轴展示**：
- 月份刻度显示在时间轴(甘特)列的**表头**上(如 7月~12月),不在每行重复
- 月份范围根据里程碑任务的开始/结束日期自动匹配
- 甘特条相对位置和宽度根据任务日期计算得出

**完成状态枚举**：未开始 / 进行中 / 已完成 / 已延期（状态为枚举值，不展示完成数量）

**交互**：点击里程碑行下钻到该任务的明细页。

#### 6.1.6 项目风险跟踪区

展示当前项目的活跃风险（状态≠已关闭），按风险等级和状态排序，帮助项目经理快速掌握风险全貌。

**表字段**：`等级 | 风险描述 | 风险影响评估 | 应对方案 | 责任人 | 登记时间 | 计划关闭时间 | 状态`

**列宽规范（固定宽度）**：等级 90px、风险描述 180px、风险影响评估 180px、应对方案 180px、责任人 120px、登记时间 140px、计划关闭时间 140px、状态 100px。

**表头与内容对齐**：全部列（表头 + 内容）均**居中显示**。适用[6.1.5](#615-里程碑区)所述通用表格规则（固定列宽、省略号截断+悬停显示、超宽横向滚动）。

**状态枚举**：已识别 / 应对中 / 已关闭

| 展示内容 | 说明                                             |
| -------- | ------------------------------------------------ |
| 筛选规则 | 首页默认仅展示活跃风险（状态 ≠ 已关闭）          |
| 排序规则 | ① 高风险 → 中风险 → 低风险（等级优先）<br>② 同级内：应对中 → 已识别（状态排序）<br>③ 同级同状态：按计划关闭时间升序 |
| 展示数量 | 默认展示 TOP 5 活跃风险，"查看全部"跳转风险管理模块 |

**交互**：点击某条风险 → 弹出风险详情抽屉；点击"查看全部" → 跳转 /risks。

#### 6.1.7 待办区

**布局**：项目待办、我的待办各自为一个**独立框**，并排展示，两框列宽一致。

**独立筛选**：每个框顶部各带一组筛选按钮（本月 / 本周 / 今日），**默认显示"本周"**。两框筛选相互独立，分别对各自的待办列表按时间范围过滤（不是对全部待办统一筛选）。

**两框内容**：

| 区域     | 权限                       | 说明                                             |
| -------- | -------------------------- | ------------------------------------------------ |
| 项目待办 | 所有用户可见（校园经理只读）| 展示未完成的**末级待办**，管理员/项目经理可指定负责人、编辑；校园经理视角下操作按钮不显示（而非置灰） |
| 我的待办 | 所有用户可见               | 当前用户的**末级待办**，支持标记完成、编辑、转办（全角色） |

**末级待办联动规则**（代码逻辑实现，页面不展示说明文字）：
- 项目待办和个人待办统一展示"末级待办"（最细颗粒度待办）的完成情况
- 当下级待办全部完成后，上级待办自动维护为已完成
- 待办支持多级树形结构，联动更新保证上级状态与下级一致

**时间筛选口径**：按待办的截止日期（due_date）判断归属本月 / 本周 / 今日。

#### 6.2.7 WBS自动生成规则

**触发条件**：设备信息模块新增设备记录时自动生成。

**仅生成建设年份≤当前年份的设备任务**（如当前2026年，只生成2026年及以前设备任务）。

**计划时间映射**：
- 完成设备到货：取设备信息的"到货计划时间"
- 完成设备安装：到货计划时间 + 3天
- 完成设备调试：安装计划时间 + 2天
- 完成教师培训：调试计划时间 + 5天

**设备编辑后再次生成规则**：
- 已有任务 → 更新计划时间，不新增
- 新增的L4任务 → 新增
- 不再需要的L4任务 → 标记为孤儿任务

**事务一致性**：设备记录保存与WBS任务生成在同一数据库事务中，全部成功或全部回滚。

#### 6.2.8 孤儿任务处理

孤儿任务在列表中特殊标记（灰色背景 + "已失效"标签），系统每周定时提醒项目经理存在孤儿任务，支持手动删除或重新关联其他设备记录。

#### 6.2.9 功能清单
- WBS树形视图（5级展开/折叠）
- 列表视图（支持筛选、**批量操作**）
- 自动生成WBS（基于设备台账）

#### 6.4.3 设备状态与WBS任务状态联动

**联动原则**：以WBS任务状态为准。WBS→设备为自动同步（主要），设备→WBS为提醒（辅助）。

**WBS→设备（自动同步）**：

| WBS任务L4内容 | 触发后设备状态变更 |
| ------------- | ------------------ |
| 完成设备到货  | → 已到货           |
| 完成设备安装  | → 已安装           |
| 完成设备调试  | → 已调试           |
| 完成设备验收  | → 已验收           |

**设备→WBS（提醒）**：手动变更设备状态时，发送站内消息提醒责任人更新WBS状态。

**一致性规则**：WBS状态回退时，设备状态不回退（物理状态不随计划回退）。

- 项目再次验收后，复盘入口重新开启，可选择"重新填写"或"基于之前内容继续完善"

#### 6.10.4 功能清单
- 复盘入口（项目状态变更为"已验收"后自动开启）
- 复盘报告编辑
- 附件上传
- 导出复盘报告（Word/PDF）

## 八、数据导入规范

### 8.1 通用规则
- 每个模块提供标准Excel模板下载
- 导入校验：必填项、数据格式、外键关联
- 错误提示：展示具体行号 + 错误原因，支持导出错误报告
- 全量校验通过后才写入

### 8.2 WBS导入增强

支持Excel导入时**自动解析L1-L5层级结构**：
- 模板列：L1阶段、L2子阶段、L3任务包、L4工作内容、L5工作明细、计划开始时间、计划结束时间、责任人账号、关联学校编码、关联阶段类型
- 系统自动识别层级关系，生成树形结构
- 支持多学校批量导入

### 8.3 设备批量导入

提供标准Excel模板，支持一次导入多条设备记录：
- 模板列：系统名称、设备名称、数量、分配学校编码、安装位置、设备来源、供应商名称、到货计划时间、备注
- 校验通过后统一提交

### 8.4 数据迁移方案

| 步骤        | 内容                                               |
| ----------- | -------------------------------------------------- |
| 1. 模板下载 | 系统提供完整导入模板                               |
| 2. 数据清洗 | 建议用户先进行去重、格式标准化                     |
| 3. 分批导入 | 支持大文件分批处理，每批建议500条                  |
| 4. 导入报告 | 导入后生成报告（成功条数、失败条数、失败原因列表） |


## 九、通知机制

### 9.1 通知类型

| 通知场景         | 触发条件                     | 接收人            | 方式                |

**邮件内容模板**：
- 标题：【GX项目交付系统】催办通知 - 缺失材料提醒
- 正文：您好，您有以下交付材料尚未上传，请及时补充：[缺失材料清单] [所属学校] [任务链接]

## 十三、术语表

| 术语       | 定义                                                         |
| ---------- | ------------------------------------------------------------ |
| 校园经理   | 项目团队内部角色，负责指定学校的项目交付执行与管理           |

| 建设年份   | 设备计划建设的年份，用于区分多年度项目                       |

| 待补材料   | WBS任务完成但佐证材料缺失的中间状态                          |
| 孤儿任务   | 由设备记录自动生成，但源设备记录已被删除的WBS任务            |
| 健康度评分 | 综合延期任务、材料缺失、风险项、里程碑延期的项目健康评价     |

| 项目复盘   | 项目验收后进行的总结与经验沉淀                               |
| 快照存储   | 报告/周报生成时复制数据存储，后续源数据变更不影响已生成内容  |
