```bash
  --restart always \

  -e MYSQL_USER=gx_user \
  -e MYSQL_PASSWORD=GX2026\!pass \
```

### 3.5 实际部署方式说明（经服务器实连核实）

**当前生产环境（124.222.151.69）的 MySQL 运行在 Docker 容器中**（容器名 `gx_mysql`，镜像 `mysql:8.0`），与本章上述 Docker 部署方式一致。

| 项目 | 实际配置 |
|------|---------|
| MySQL 运行方式 | Docker 容器 `gx_mysql`（mysql:8.0） |
| 启动/停止 | `docker start/stop gx_mysql` |
| 连接方式 | `docker exec -it gx_mysql mysql -uroot -pGX2026\!root` |
| 端口映射 | 宿主机 3306 → 容器 3306（docker-proxy 转发） |
| 后端连接 | 后端 `.env` 用 `DB_HOST=localhost`（经端口映射连到容器） |
| 数据库 | `gx_project`（生产）+ `gx_project_dev`（开发） |

**验证实际部署方式：**
```bash

# 查看 3306 监听者（应为 docker-proxy）

```

> **注意**：本机 `systemctl status mysql` 为 inactive 属正常现象——MySQL 不在本机运行，而在 Docker 容器内。运维时统一使用 `docker` 命令管理，不要用 `systemctl mysql`。

## 六、配置 Nginx

### 6.1 安装 Nginx

```bash
sudo apt install -y nginx
```

### 6.2 站点配置（当前生产环境实际配置）

当前生产环境使用 `/etc/nginx/sites-enabled/default`（监听 80 端口，root 直接指向 build 目录），配置如下：

```nginx
server {
    listen 80 default_server;
    server_name _;

    # 安全：隐藏 server 版本
    server_tokens off;

    # 前端静态文件（直接指向 build 目录）
    root /home/ubuntu/gx-project/frontend/build;
    index index.html;

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 7d;
        add_header Cache-Control "public, max-age=604800, immutable";
        try_files $uri =404;
    }

    # SPA 路由 fallback（react-router 用）
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 反代
    location /api/ {
        proxy_pass http://127.0.0.1:8000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 60s;
        proxy_connect_timeout 10s;
    }

    # 健康检查直连后端
    location = /health {
        proxy_pass http://127.0.0.1:8000/health;
    }

    # 安全：禁掉敏感路径扫描
    location ~* /(SDK|developmentserver|actuator|env|\.env|\.git) {
        return 444;
    }
}
```


## 十、常见问题

### 10.1 端口被占用

```bash
sudo lsof -i :8000
sudo kill -9 <PID>
sudo systemctl start gx-backend
```

### 10.2 MySQL 容器无法启动

```bash
docker logs gx_mysql
# 查看具体错误原因
```

### 10.3 后端无法连接 MySQL

检查 `.env` 中的 `DB_HOST` 配置，参考第七章”多环境数据库连接说明”。

### 10.4 本地开发环境连不上远程数据库（UFW 防火墙）

**现象**：本地后端启动后报 `(2003, “Can't connect to MySQL server on '124.222.151.69:3306'”)`

**原因**：服务器已启用 UFW 防火墙，默认只放行 22/80/443 端口，3306 端口仅对白名单 IP 开放。

**解决方案**：
1. 确认本机公网 IP（访问 https://api.ipify.org 或 `curl ifconfig.me`）
2. 联系服务器管理员将该 IP 加入 UFW 全端口白名单：
   ```bash
   sudo ufw allow from <你的公网IP> to any
   sudo ufw reload
   ```
3. 若经常切换网络（WiFi/热点/VPN），公网 IP 会变化，需重新加白名单

**临时测试连通性**：
```bash
# Windows PowerShell
Test-NetConnection -ComputerName 124.222.151.69 -Port 3306

# Linux/Mac
telnet 124.222.151.69 3306
```

### 10.5 前端跨域错误（CORS）

**现象**：浏览器控制台报 `Access to XMLHttpRequest ... blocked by CORS policy: No 'Access-Control-Allow-Origin' header`

**原因**：前端访问地址（如 `http://127.0.0.1:3000`）未在后端 CORS 白名单内。

**解决方案**：
1. 检查后端 `app/main.py` 的 `allow_origins` 配置
2. 确保包含前端实际访问地址：
   ```python
   allow_origins=[
       “http://localhost:3000”,
       “http://127.0.0.1:3000”,  # ← 必须包含
   ]
   ```
3. 修改后**必须重启后端**才生效：`sudo systemctl restart gx-backend`

**注意**：`localhost` 和 `127.0.0.1` 对浏览器是不同源，必须同时加入白名单。

### 10.6 生产前端更新后仍显示旧内容

**原因**：生产环境用的是静态 build 模式，代码更新需重新构建。

**完整更新流程**：
```bash
cd ~/gx-project/frontend
git pull                          # 拉取代码
npm install --legacy-peer-deps    # 更新依赖（若 package.json 变更）
npm run build                     # 重新构建（Nginx 直接读取 build 目录，无需复制）
sudo systemctl reload nginx       # 重载 Nginx（可选）
```

**浏览器缓存清理**：访问时按 `Ctrl+Shift+R` 强制刷新。
