# GX教育项目交付管理系统 - 部署手册

**版本**：V1.1
**日期**：2026年7月


## 一、环境要求

| 组件 | 版本 | 说明 |
|------|------|------|
| Ubuntu | 24.04 LTS | 服务器操作系统 |
| Docker | 20.10+ | 容器运行时 |
| Python | 3.11+ | 后端运行环境 |
| Node.js | 18.x LTS | 前端构建环境 |
| MySQL | 8.0 | 数据库（Docker容器） |
| Nginx | 最新 | 反向代理（生产环境） |


## 二、服务器初始化

### 2.1 系统更新

```bash
sudo apt update && sudo apt upgrade -y
```

### 2.2 安装必要工具

```bash
sudo apt install -y git curl wget net-tools screen
```

### 2.3 安装 Docker

```bash
# 安装 Docker
sudo apt install -y docker.io docker-compose-plugin

# 启动并设置开机自启
sudo systemctl start docker
sudo systemctl enable docker

# 将当前用户加入 docker 组（避免每次使用 sudo）
sudo usermod -aG docker $USER

# 验证
docker --version
```

### 2.4 安装 Python

```bash
sudo apt install -y python3 python3-pip python3-venv
```

### 2.5 安装 Node.js 18

```bash
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt install -y nodejs
node --version
```


## 三、部署 MySQL

### 3.1 创建数据目录

```bash
mkdir -p ~/gx-project/mysql-data
```

### 3.2 启动 MySQL 容器

```bash
docker run -d \
  --name gx_mysql \
  --restart always \
  -p 3306:3306 \
  -e MYSQL_ROOT_PASSWORD=GX2026\!root \
  -e MYSQL_DATABASE=gx_project \
  -e MYSQL_USER=gx_user \
  -e MYSQL_PASSWORD=GX2026\!pass \
  -v ~/gx-project/mysql-data:/var/lib/mysql \
  mysql:8.0 \
  --character-set-server=utf8mb4 \
  --collation-server=utf8mb4_unicode_ci
```

### 3.3 验证 MySQL 运行状态

```bash
docker ps | grep gx_mysql
```

### 3.4 创建开发数据库（可选）

```bash
docker exec -i gx_mysql mysql -uroot -pGX2026\!root -e "CREATE DATABASE IF NOT EXISTS gx_project_dev CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
docker exec -i gx_mysql mysql -uroot -pGX2026\!root gx_project_dev < ~/gx-project/ddl.sql
```


## 四、部署后端

### 4.1 创建项目目录

```bash
mkdir -p ~/gx-project/backend
cd ~/gx-project
```

### 4.2 克隆代码（如果有 Git 仓库）

```bash
git clone <仓库地址> .
# 或直接上传代码
```

### 4.3 创建 Python 虚拟环境

```bash
cd ~/gx-project/backend
python3 -m venv venv
source venv/bin/activate
```

### 4.4 依赖清单（requirements.txt）

后端 Python 依赖统一由 `requirements.txt` 管理，文件位于 `~/gx-project/backend/requirements.txt`：

```txt
# FastAPI 核心
fastapi==0.109.0
uvicorn[standard]==0.27.0

# 数据库
sqlalchemy==2.0.25
pymysql==1.1.0

# 认证与加密
python-jose[cryptography]==3.3.0
passlib[bcrypt]==1.7.4
bcrypt==4.1.2

# 环境变量
python-dotenv==1.0.0

# 数据校验
pydantic==2.5.3

# 表单数据解析
python-multipart==0.0.6

# HTTP 客户端（开发测试用）
httpx==0.27.0
```

### 4.5 安装依赖

```bash
cd ~/gx-project/backend
source venv/bin/activate
pip install -r requirements.txt
```

### 4.6 配置环境变量

创建 `.env` 文件：

```bash
cat > ~/gx-project/backend/.env << 'EOF'
DB_HOST=localhost
DB_PORT=3306
DB_USER=root
DB_PASSWORD=GX2026!root
DB_NAME=gx_project
SECRET_KEY=请更换此密钥（生成方法：openssl rand -hex 32）
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=1440
EOF
```

> **SECRET_KEY 生成方法**：`openssl rand -hex 32`

### 4.7 初始化数据库

```bash
docker exec -i gx_mysql mysql -uroot -pGX2026\!root gx_project < ~/gx-project/ddl.sql
```

### 4.8 创建 systemd 服务

```bash
sudo tee /etc/systemd/system/gx-backend.service << 'EOF'
[Unit]
Description=GX Project Backend Service
After=network.target docker.service
Wants=docker.service

[Service]
Type=simple
User=ubuntu
WorkingDirectory=/home/ubuntu/gx-project/backend
Environment="PATH=/home/ubuntu/gx-project/backend/venv/bin:/usr/local/bin:/usr/bin:/bin"
ExecStart=/home/ubuntu/gx-project/backend/venv/bin/uvicorn app.main:app --host 127.0.0.1 --port 8000
Restart=always
RestartSec=10
StandardOutput=journal
StandardError=journal

[Install]
WantedBy=multi-user.target
EOF
```

### 4.9 启动后端服务

```bash
sudo systemctl daemon-reload
sudo systemctl enable gx-backend
sudo systemctl start gx-backend
sudo systemctl status gx-backend
```

### 4.10 查看后端日志

```bash
sudo journalctl -u gx-backend -f
```


## 五、部署前端

### 5.1 进入前端目录

```bash
cd ~/gx-project/frontend
```

### 5.2 安装依赖

```bash
npm install --legacy-peer-deps
```

### 5.3 构建生产版本

```bash
npm run build
```

构建产物在 `build/` 目录。

### 5.4 部署静态文件

```bash
sudo mkdir -p /var/www/gx
sudo cp -r build/* /var/www/gx/
```

### 5.5 前端环境变量配置

前端支持多环境配置，通过 `.env` 文件管理 API 地址：

| 文件               | 用途         | API地址                      |
| ------------------ | ------------ | ---------------------------- |
| `.env.development` | 本地开发环境 | `http://127.0.0.1:8000`      |
| `.env.production`  | 生产环境     | `http://124.222.151.69:8000` |

> 注意：`REACT_APP_API_URL` 在构建时注入，修改后需要重新构建前端。

## 六、配置 Nginx

### 6.1 安装 Nginx

```bash
sudo apt install -y nginx
```

### 6.2 创建站点配置

```bash
sudo tee /etc/nginx/sites-available/gx-project << 'EOF'
server {
    listen 8080;
    server_name _;

    root /var/www/gx;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
EOF
```

### 6.3 启用站点

```bash
sudo ln -s /etc/nginx/sites-available/gx-project /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl restart nginx
```

### 6.4 验证访问

浏览器访问 `http://<服务器IP>:8080`，应看到登录页面。


## 七、多环境数据库连接说明

| 场景                     | DB_HOST 配置               | 说明                                   |
| ------------------------ | -------------------------- | -------------------------------------- |
| 后端容器内访问 MySQL     | `localhost` 或 `127.0.0.1` | 宿主机与容器在同一网络                 |
| 宿主机访问容器内 MySQL   | `127.0.0.1`                | 端口映射到宿主机                       |
| 远程开发访问服务器 MySQL | `服务器公网IP`             | 需开放 3306 端口（不建议生产环境开放） |
| 本地开发访问服务器 MySQL | `124.222.151.69`           | 需开放 3306 端口                       |

```bash
sudo lsof -i :8000

```

### 10.2 MySQL 容器无法启动

```bash
docker logs gx_mysql
```
