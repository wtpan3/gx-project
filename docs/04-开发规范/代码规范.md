## 一、总体原则

1. **可读性优先**：代码是写给团队成员看的，其次才是给机器执行的

    验证用户登录
    
    Args:
        db: 数据库会话
        username: 用户名

### 3.5 API 调用规范

#### 环境变量配置

前端通过 `.env` 文件管理不同环境的 API 地址：

```bash
# .env.development（本地开发）
REACT_APP_API_URL=http://127.0.0.1:8000

# .env.production（生产环境）
REACT_APP_API_URL=http://124.222.151.69:8000
```

> **使用规则**：`npm start` 自动加载 `.env.development`，`npm run build` 自动加载 `.env.production`。

#### 统一 API 客户端

创建 `frontend/src/services/api.ts`：

```typescript
import axios from 'axios';

const api = axios.create({
  baseURL: process.env.REACT_APP_API_URL || 'http://127.0.0.1:8000',
  timeout: 30000,
});

// 请求拦截器
api.interceptors.request.use((config) => {
  const token = localStorage.getItem('token');
  if (token) {
    config.headers.Authorization = `Bearer ${token}`;
  }
  return config;
});

// 响应拦截器
api.interceptors.response.use(
  (response) => response,
  (error) => {
    if (error.response?.status === 401) {
      // 跳转到登录页
      window.location.href = '/login';
    }
    return Promise.reject(error);
  }
);

export default api;
```

### 3.6 UI 布局与表格规范（强制）

**核心原则：宽度不足时横向滚动，绝不压缩内容或强制换行。**

适用于所有页面的表格、卡片行、多栏并排区域（含首页卡片区、交付进度区、里程碑、风险跟踪等，以及后续新增页面）。

#### 3.6.1 容器溢出规则

当内容（表格、卡片组、多栏布局）的总宽度超出父容器时，给容器增加 `overflow-x: auto`，允许横向滚动查看完整内容，**而不是压缩列宽/卡片宽度或强制换行**。

```css
/* 通用横向滚动容器 */
.scroll-x { overflow-x: auto; }
/* 内层设 min-width 保证子项不被挤压 */
.scroll-x > .row { display: flex; gap: 12px; min-width: 980px; }
.scroll-x > .row > .item { flex: 1; min-width: 120px; }
```

#### 3.6.2 表格规范

1. 采用 `table-layout: fixed`，每一列设置**固定宽度**，不允许内容自动撑开列宽
2. 单元格文本超出列宽时用省略号（…）截断：`white-space: nowrap; overflow: hidden; text-overflow: ellipsis;`
3. 被截断的单元格需可查看全文：加 `title` 属性或 Ant Design `<Tooltip>` 组件
4. 表格设 `min-width`（各列固定宽度之和），外层容器 `overflow-x: auto`，超宽时横向滚动
5. 表头与内容对齐方式保持一致（如首页里程碑、风险表统一居中）

```css
.tbl-container { overflow-x: auto; }
.tbl { width: 100%; table-layout: fixed; min-width: 1060px; }
.tbl th, .tbl td { white-space: nowrap; overflow: hidden; text-overflow: ellipsis; text-align: center; }
```

```tsx
// Ant Design Table：固定列宽 + 省略 + 悬停 + 横向滚动
<Table
  columns={[
    { title: '任务', dataIndex: 'task', width: 180, ellipsis: { showTitle: true }, align: 'center' },
    // ...其余列均设 width
  ]}
  scroll={{ x: 1060 }}   // 总宽超容器时横向滚动，不压缩列
  tableLayout="fixed"
/>
```

#### 3.6.3 卡片行 / 多栏布局

- 卡片行、多栏并排区域用 flex 布局，容器加 `overflow-x: auto`，内层 `min-width` 兜底
- 每个卡片/栏目设 `min-width`，宽度不足时整体横向滚动，不挤压单卡内容
- 卡片内的标签、下钻链接等短文本加 `white-space: nowrap`，避免折行


## 四、数据库规范

### 4.1 表命名

| 类型   | 规范                    | 示例                       |
| ------ | ----------------------- | -------------------------- |
| 表名   | 小写 + 下划线，复数形式 | `users`, `device_systems`  |
| 字段名 | 小写 + 下划线           | `created_at`, `updated_at` |
| 主键   | `id`                    | 自增整数                   |
| 外键   | 关联表名 + `_id`        | `school_id`, `user_id`     |

### 4.2 常用字段约定

| 字段       | 类型                                                         | 说明     |
| ---------- | ------------------------------------------------------------ | -------- |
| id         | INT PRIMARY KEY AUTO_INCREMENT                               | 主键     |
| created_at | DATETIME DEFAULT CURRENT_TIMESTAMP                           | 创建时间 |
| updated_at | DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |
| status     | ENUM('启用','停用')                                          | 状态字段 |
| is_enabled | TINYINT DEFAULT 1                                            | 启用标记 |

### 4.3 枚举字段

优先使用 `ENUM` 类型定义固定选项：

```sql
role ENUM('admin','project_manager','campus_manager') NOT NULL
status ENUM('启用','停用') DEFAULT '启用'
```
