## 一、服务器安全

### 1.1 SSH 安全配置

编辑 `/etc/ssh/sshd_config`：

```bash
# 禁用 root 登录
PermitRootLogin no

# 禁用密码登录（仅允许密钥）
PasswordAuthentication no

# 修改默认端口（可选）
Port 2222

# 限制登录尝试次数
MaxAuthTries 3

# 重启 SSH 服务
sudo systemctl restart sshd
```

### 1.2 防火墙配置（UFW）

```bash
# 启用 UFW
sudo ufw enable

# 仅开放必要端口
sudo ufw allow 22/tcp      # SSH（如修改了端口，请对应调整）
sudo ufw allow 80/tcp      # HTTP
sudo ufw allow 443/tcp     # HTTPS
sudo ufw allow 8080/tcp    # 应用访问
# 注意：8000 端口不应直接对外暴露，仅内网访问

# 查看状态
sudo ufw status
```

### 1.3 腾讯云安全组建议

| 端口 | 建议策略     | 说明                                   |
| ---- | ------------ | -------------------------------------- |
| 22   | 限制来源IP   | 仅允许公司IP或跳板机IP访问             |
| 80   | 允许所有     | HTTP 访问（建议重定向到 HTTPS）        |
| 443  | 允许所有     | HTTPS 访问                             |
| 8080 | 允许所有     | 应用访问（可考虑仅允许内网+Nginx代理） |
| 8000 | **禁止开放** | 后端 API 仅内网访问，通过 Nginx 代理   |
| 3306 | **禁止开放** | MySQL 仅内网访问                       |


## 二、应用安全

### 2.1 JWT 配置

```python
# 使用强密钥（至少32字节）
SECRET_KEY = os.getenv("SECRET_KEY")  # 通过环境变量注入，不硬编码

# 设置合理的过期时间
ACCESS_TOKEN_EXPIRE_MINUTES = 60 * 8   # 8小时
REFRESH_TOKEN_EXPIRE_DAYS = 7          # 7天
```

### 2.2 密码策略

| 要求     | 说明                       |
| -------- | -------------------------- |
| 最小长度 | 8 位                       |
| 复杂度   | 至少包含数字和字母         |
| 存储方式 | bcrypt 加密（不可逆）      |
| 重置方式 | 通过管理员重置或邮件验证码 |

### 2.3 SQL 注入防护

- 使用 SQLAlchemy ORM（参数化查询）
- 禁止拼接 SQL 字符串
- 危险字符过滤

```python
# 错误做法
query = f"SELECT * FROM users WHERE username = '{username}'"

# 正确做法
query = db.query(User).filter(User.username == username)
```

### 2.4 XSS 防护

- 前端使用 React 默认转义（`{}` 输出自动转义）
- 避免使用 `dangerouslySetInnerHTML`
- 对用户输入进行验证和清理

### 2.5 CORS 配置

```python
# 生产环境限制允许的来源
app.add_middleware(
    CORSMiddleware,
    allow_origins=["https://your-domain.com"],  # 禁止使用 "*"
    allow_credentials=True,
    allow_methods=["GET", "POST", "PUT", "DELETE"],
    allow_headers=["Authorization", "Content-Type"],
)
```

### 2.6 敏感字段保护

以下字段不应出现在 API 响应中：

- `password_hash`
- `token`
- 用户手机号（非必要不返回）
- 邮箱（非必要不返回）


## 三、数据安全

### 3.1 数据库访问限制

- MySQL 端口不对外开放
- 使用专用数据库账号（非 root）
- 定期更换数据库密码

### 3.2 数据备份

详见《灾备与恢复方案.md》

### 3.3 敏感信息脱敏

日志中不记录敏感信息：

```python
# 日志记录时避免输出密码、token等
logger.info(f"User login: {username}")  # 不记录密码
```
