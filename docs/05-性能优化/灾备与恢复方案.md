## 一、备份策略总览

| 备份类型 | 频率 | 保留周期 | 内容 |
|----------|------|----------|------|
| 数据库全量备份 | 每日凌晨3:00 | 30天 | MySQL所有数据 |
| 数据库增量备份 | 每6小时 | 7天 | 二进制日志 |
| 文件备份 | 每日凌晨4:00 | 30天 | 上传的文件（交付材料库） |
| 系统配置备份 | 每次变更后 | 永久 | Nginx、systemd、环境变量 |


## 二、数据库备份

### 2.1 手动全量备份

```bash
# 创建备份目录
mkdir -p ~/backup

# 备份生产库
docker exec gx_mysql mysqldump -uroot -pGX2026\!root \
  --single-transaction \
  --routines \
  --triggers \
  --databases gx_project > ~/backup/gx_project_$(date +%Y%m%d_%H%M%S).sql

# 备份开发库
docker exec gx_mysql mysqldump -uroot -pGX2026\!root \
  --single-transaction \
  gx_project_dev > ~/backup/gx_project_dev_$(date +%Y%m%d_%H%M%S).sql
```

### 2.2 自动备份脚本

创建 `~/backup/backup.sh`：

```bash
#!/bin/bash
# 数据库自动备份脚本

BACKUP_DIR=/home/ubuntu/backup
DATE=$(date +%Y%m%d_%H%M%S)
RETENTION_DAYS=30

# 备份
docker exec gx_mysql mysqldump -uroot -pGX2026\!root \
  --single-transaction \
  --routines \
  --triggers \
  gx_project > $BACKUP_DIR/gx_project_$DATE.sql

docker exec gx_mysql mysqldump -uroot -pGX2026\!root \
  --single-transaction \
  gx_project_dev > $BACKUP_DIR/gx_project_dev_$DATE.sql

# 压缩
gzip $BACKUP_DIR/gx_project_$DATE.sql
gzip $BACKUP_DIR/gx_project_dev_$DATE.sql

# 删除过期备份（保留30天）
find $BACKUP_DIR -name "*.sql.gz" -mtime +$RETENTION_DAYS -delete

echo "Backup completed: $DATE"
```

设置定时任务：

```bash
chmod +x ~/backup/backup.sh
crontab -e
# 添加以下行（每天凌晨3点执行）
0 3 * * * /home/ubuntu/backup/backup.sh >> /home/ubuntu/backup/backup.log 2>&1
```

### 2.3 备份文件下载到本地

```bash
# 在本地执行
scp -i ~/.ssh/gx.pem ubuntu@124.222.151.69:~/backup/gx_project_*.sql.gz ./
```


## 三、文件备份

### 3.1 上传文件备份

```bash
# 创建备份脚本
cat > ~/backup/files_backup.sh << 'EOF'
#!/bin/bash
BACKUP_DIR=/home/ubuntu/backup/files
DATE=$(date +%Y%m%d)
mkdir -p $BACKUP_DIR

# 打包并压缩上传文件
tar -czf $BACKUP_DIR/upload_files_$DATE.tar.gz ~/gx-project/uploads/ 2>/dev/null || true

# 删除30天前的备份
find $BACKUP_DIR -name "*.tar.gz" -mtime +30 -delete
EOF

chmod +x ~/backup/files_backup.sh
```


## 四、系统配置备份

### 4.1 一键备份所有配置

```bash
cat > ~/backup/config_backup.sh << 'EOF'
#!/bin/bash
BACKUP_DIR=/home/ubuntu/backup/configs
DATE=$(date +%Y%m%d)
mkdir -p $BACKUP_DIR

# 备份 Nginx 配置
sudo cp -r /etc/nginx/sites-available $BACKUP_DIR/nginx_$DATE/
sudo cp -r /etc/nginx/sites-enabled $BACKUP_DIR/nginx_$DATE/

# 备份 systemd 服务文件
sudo cp /etc/systemd/system/gx-backend.service $BACKUP_DIR/

# 备份 .env 文件
cp ~/gx-project/backend/.env $BACKUP_DIR/backend_env_$DATE

# 备份 crontab
crontab -l > $BACKUP_DIR/crontab_$DATE 2>/dev/null

echo "Config backup completed: $DATE"
EOF

chmod +x ~/backup/config_backup.sh
```


## 五、数据恢复

### 5.1 数据库恢复

```bash
# 1. 停止后端服务（避免写入）
sudo systemctl stop gx-backend

# 2. 恢复数据库
docker exec -i gx_mysql mysql -uroot -pGX2026\!root < ~/backup/gx_project_20260101_030000.sql

# 3. 重启后端
sudo systemctl start gx-backend
```

### 5.2 从备份恢复指定表

```bash
# 只恢复 users 表
docker exec -i gx_mysql mysql -uroot -pGX2026\!root gx_project -e "DROP TABLE users;"
docker exec -i gx_mysql mysql -uroot -pGX2026\!root gx_project < ~/backup/users_restore.sql
```

### 5.3 文件恢复

```bash
# 解压备份文件
tar -xzf ~/backup/files/upload_files_20260101.tar.gz -C ~/gx-project/uploads/
```


## 六、灾难恢复计划

### 6.1 服务器完全损坏

**场景**：CVM 无法启动或数据全部丢失

**恢复步骤**：

1. **重建服务器**（新 CVM）
2. **重新部署环境**（参考部署手册）
3. **恢复数据库**（从最新备份恢复）
4. **恢复文件**（从最新文件备份恢复）
5. **恢复配置**（Nginx、systemd、.env）

### 6.2 数据库损坏

**场景**：MySQL 数据损坏，容器无法启动

**恢复步骤**：

```bash
# 1. 停止损坏的容器
docker stop gx_mysql

# 2. 移除数据目录
sudo rm -rf ~/gx-project/mysql-data

# 3. 重新创建容器（参考部署手册）
docker run -d --name gx_mysql ...

# 4. 恢复数据
docker exec -i gx_mysql mysql -uroot -pGX2026\!root < ~/backup/gx_project_最新.sql

# 5. 验证数据
docker exec -it gx_mysql mysql -uroot -pGX2026\!root -e "USE gx_project; SELECT COUNT(*) FROM users;"
```

### 6.3 人为误操作数据删除

**场景**：用户误删重要数据

**恢复步骤**：

```bash
# 1. 立即停止后端服务
sudo systemctl stop gx-backend

# 2. 从最近的备份恢复
docker exec -i gx_mysql mysql -uroot -pGX2026\!root gx_project < ~/backup/gx_project_最近时间.sql

# 3. 确认数据完整后重启
sudo systemctl start gx-backend
```
