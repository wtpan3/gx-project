# GX教育项目交付管理系统 - 测试规范

**版本**: V1.0
**日期**: 2026-07-21
**适用**: 开发阶段测试、交付验收测试


## 一、测试类型定义

| 测试类型     | 说明                              | 执行人        | 执行时机       |
| ------------ | --------------------------------- | ------------- | -------------- |
| **单元测试** | 单个API接口、函数、组件的功能验证 | 开发者        | 开发过程中     |
| **集成测试** | 前后端联调、数据库交互验证        | 开发者        | 模块开发完成后 |
| **功能测试** | 按需求文档逐功能验证              | 测试者/产品   | 每批次交付前   |
| **回归测试** | 已有功能不受新代码影响            | 测试者        | 每次代码变更后 |
| **验收测试** | 项目经理对交付成果进行验收        | 项目经理/用户 | 批次交付时     |


## 二、测试执行时机

| 阶段       | 测试类型            | 说明                   |
| ---------- | ------------------- | ---------------------- |
| 开发中     | 单元测试 + 集成测试 | 开发者自测             |
| 代码提交前 | 集成测试            | 确认不影响已有功能     |
| 批次交付前 | 功能测试 + 回归测试 | 按验收标准清单逐项验证 |
| 批次交付时 | 验收测试            | 项目经理签字确认       |


## 三、测试环境

| 环境     | 说明          | 数据库                      |
| -------- | ------------- | --------------------------- |
| 本地开发 | 开发者本地 PC | 远程开发库 `gx_project_dev` |

| 生产环境 | 正式部署      | 远程生产库 `gx_project`     |
