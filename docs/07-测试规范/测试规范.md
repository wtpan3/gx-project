## 一、测试类型定义

| 测试类型     | 说明                              | 执行人        | 执行时机       |
| ------------ | --------------------------------- | ------------- | -------------- |
| **单元测试** | 单个API接口、函数、组件的功能验证 | 开发者        | 开发过程中     |
| **集成测试** | 前后端联调、数据库交互验证        | 开发者        | 模块开发完成后 |
| **功能测试** | 按需求文档逐功能验证              | 测试者/产品   | 每批次交付前   |
| **回归测试** | 已有功能不受新代码影响            | 测试者        | 每次代码变更后 |
| **部署后端到端测试** | 真实/无头浏览器走完整用户路径，验证部署真的可用 | 部署执行者（含Claude） | **每次部署后（强制）** |
| **验收测试** | 项目经理对交付成果进行验收        | 项目经理/用户 | 批次交付时     |

## 五、真实场景端到端测试（强制，开发与部署通用）

> 🔑 **总原则（Definition of Done）**：任何工作——无论是**开发环境改完一个功能**，还是**部署到生产**——都必须用**真实使用场景**在浏览器里走通，才能称为"完成"。
> 这不是只针对部署的要求。**开发环境同样适用**：改完代码不等于完成，要在浏览器里像真实用户那样操作走通、确认无误，才算完成。

> ⚠️ **来自真实教训**：曾多次出现"curl 测接口返回 200、宣布完成，但用户浏览器登录失败/白屏"。
> 根因：**curl 绕过了浏览器行为**（CORS、JS 执行、登录后的连锁请求），curl 通过 ≠ 用户可用。

### 5.0 适用节点（两个都强制）

| 节点 | 环境 | 端到端验证要求 |
|------|------|---------------|
| **开发完成一个功能/修改** | 开发环境（localhost:3000） | 浏览器打开对应页面，实际操作该功能走通，F12 无报错，才能标记该功能"开发完成" |
| **部署/更新到生产** | 生产环境（124.222.151.69） | 浏览器走完整用户路径（登录→首页→核心页面），才能宣布"部署完成" |

**改了接口，不能只测接口本身，必须测调用它的前端页面在浏览器里是否正常。**

### 5.1 验证方法分级（必须区分并如实报告）

| 级别 | 方法 | 能发现 | 结论表述 |
|------|------|--------|---------|
| L1 服务存活 | `systemctl` / `docker ps` | 进程在跑 | 只能说"服务已启动" |
| L2 接口连通 | `curl` 打单接口 | 接口可达 | 只能说"接口可达"，**发现不了 CORS** |
| L3 端到端 ✅ | **浏览器走完整路径** | CORS、白屏、连锁500、前端报错 | 通过后才可说"完成、可使用"（开发/部署同标准） |

### 5.2 端到端必测清单

**A. 开发完成时（开发环境 localhost:3000）**
```
[ ] 浏览器打开开发前端，本次改动的页面正常渲染（非白屏）
[ ] F12 控制台无红色报错
[ ] 实际操作本次开发/修改的功能，完整走通（增删改查、跳转、下钻等）
[ ] 涉及的接口在页面上真实触发并返回正确结果（不是单独 curl）
[ ] 登录态、权限相关改动，换对应角色账号验证
```

**B. 部署完成时（生产环境 124.222.151.69）**
```
[ ] 浏览器打开生产地址，首页正常渲染（非白屏）
[ ] F12 控制台无红色报错（重点：CORS / Network Error）
[ ] 实际登录成功进入系统
[ ] 登录后 Dashboard 及连锁接口无 500
[ ] 核心页面逐个点开无报错
[ ] 非管理员账号权限正常
```

### 5.3 推荐工具

用 `playwright-ui-auto` skill 做无头浏览器自动化验证——真实浏览器环境，自动捕获 CORS、白屏、连锁 500，比人工点更可靠、可复现。

### 5.4 报告纪律（开发汇报"完成"、部署汇报"完成"均适用）

- 必须区分"L2 接口验证"与"L3 端到端验证"，不得用"测试通过""完成了"笼统概括
- 有未验证项必须明说，禁止在只做了 L1/L2 的情况下宣布"完成 / 可以正常使用"
- "代码写完"只能说"代码写完，待端到端验证"，不得直接说"功能完成"
