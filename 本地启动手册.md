# GX项目本地开发环境启动手册

## 前置准备

### 1. 安装 Docker Desktop（Windows）

**官方下载**：https://www.docker.com/products/docker-desktop/

**国内镜像**（更快）：
- 阿里云：https://developer.aliyun.com/mirror/docker-ce
- 腾讯云：https://mirrors.cloud.tencent.com/docker-ce/

下载后双击安装，重启电脑，确认Docker启动成功（任务栏右下角有Docker图标）。

### 2. 确认环境

```bash
# 检查版本
python --version    # 已安装 Python 3.12.8
node --version      # 已安装 Node v24.13.1
docker --version    # 安装Docker后执行
```

## 启动步骤

### 第一步：启动 MySQL

```bash
cd "f:/claude code/Projectmanage/gx-project"
docker-compose up -d
```

等待 MySQL 容器启动完成（约 30 秒），确认状态：

```bash
docker ps
# 应该看到 gx_mysql 容器 STATUS 为 Up
```

### 第二步：初始化数据库

```bash
cd backend
python init_db.py
```

看到提示 "✓ 种子数据创建完成"，说明成功。

**默认账号**：
- 管理员：admin / Admin@2026
- 项目经理：pm001 / Admin@2026

### 第三步：启动后端

```bash
# 在 backend 目录下
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

看到 `Application startup complete` 说明后端启动成功。

浏览器访问 http://localhost:8000/health 应该看到 `{"status":"ok"}`。

### 第四步：启动前端

**新开一个终端窗口**：

```bash
cd "f:/claude code/Projectmanage/gx-project/frontend"

# 首次运行需要安装依赖
npm install

# 启动开发服务器
npm start
```

等待编译完成，浏览器会自动打开 http://localhost:3000，看到登录页面。

## 常见问题

### Docker 启动失败

- 确认 Docker Desktop 已启动（任务栏有图标）
- 确认端口 3306 未被占用：`netstat -ano | findstr :3306`

### 后端启动报错 "No module named 'app'"

确认在 `backend` 目录下执行 `uvicorn` 命令。

### 前端登录失败

1. 确认后端已启动（访问 http://localhost:8000/health）
2. 打开浏览器开发者工具（F12）查看 Network 标签页，检查请求是否正常

## 停止服务

```bash
# 停止 MySQL
docker-compose down

# 停止后端和前端：各自终端按 Ctrl+C
```
